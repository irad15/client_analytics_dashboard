# Building a Dynamic Dashboard + Data Correction - Part B

The CEO of Migdal Financial Services wants a tool that allows him to view customer data from various angles and get a real-time overview. Additionally, a system issue was discovered: the client numbers (`client_id`) do not match the order of their join dates - meaning, a client with a lower number did not necessarily join before a client with a higher number.

Data file (`clients_data.csv`) with 2000 records of company clients. The data includes:
* **Client Details** - age, gender, city, join date.
* **Financial Information** - service type, portfolio amount, monthly revenue to the company.
* **Activity** - number of inquiries, average response time, satisfaction (1-10).
* **Status** - active/inactive, churn date (if applicable).

### Task A - Dynamic Dashboard for the CEO
Build an interactive dashboard that allows the CEO to view the data from different perspectives.
* The dashboard must include at least: filtering by city / service type / status, graphical display of trends (churn over time, satisfaction distribution), and the ability to explore the data.
* **Important:** The dashboard needs to be useful and intuitive - imagine the CEO is non-technical and wants to get answers with a click.

### Task B - Correcting Client Number Order
As mentioned, the client numbers (`client_id`) do not match the join order. The company currently has no ability to fix the system itself.
* Add a button to the dashboard that, when clicked, generates an updated Excel file where the client numbers are reordered according to the chronological order of the join dates (the client who joined first receives C1000, the second C1001, etc.).
* The exported file must retain all other data for each client - only the `client_id` changes.
